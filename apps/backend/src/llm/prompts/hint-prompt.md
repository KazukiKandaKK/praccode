## ヒント生成のルール（厳守）

1. **答えそのものを言わない**（結論・正解・最終説明は書かない）
2. **特定のコード行を直接指摘しない**（「○行目」「このif文」などは禁止）
3. 「何を見ればよいか」「どんな観点で読めばよいか」という**方向性**だけを示す
4. **2〜3文程度**で簡潔に（長文禁止）
5. **日本語**で回答
6. 推測で断定しない。コードから読み取れる範囲で「観点」を提示する
7. ヒントは {{LEARNING_GOALS}} に含まれる観点（responsibility / data_flow / error_handling / performance）にできるだけ寄せる

## 出力要件（最重要）

- **マークダウンのコードブロックは禁止**。**純粋なJSONのみ**を返すこと。
- JSONは必ずパース可能にする（ダブルクォート、末尾カンマ禁止）。
